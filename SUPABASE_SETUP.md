# Supabase Integration Guide

## Step 1: Create a Supabase Account

1. Go to https://supabase.com
2. Click "Start your project"
3. Create a new project
4. Wait for the project to be initialized

## Step 2: Create the Database Table

In your Supabase dashboard:

1. Go to **SQL Editor**
2. Click **New Query**
3. Paste this SQL:

```sql
CREATE TABLE attendance (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  name TEXT NOT NULL,
  roll_no TEXT NOT NULL,
  department TEXT,
  timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Create index on roll_no for faster queries
CREATE INDEX idx_attendance_roll_no ON attendance(roll_no);
CREATE INDEX idx_attendance_timestamp ON attendance(timestamp);
```

4. Click **Run**

## Step 3: Get Your Credentials

In your Supabase project dashboard:

1. Click **Settings** (bottom of left sidebar)
2. Click **API** 
3. Copy these values:
   - **Project URL** → `SUPABASE_URL`
   - **anon public** key → `SUPABASE_KEY`
   - **service_role secret** key → `SUPABASE_ADMIN_KEY` (optional, for admin tasks)

## Step 4: Create .env File

1. In your project directory, create a file named `.env`
2. Copy this template:

```bash
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_KEY=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

Replace:
- `your-project-id` with your actual project ID
- The key with your actual anon public key

**⚠️ Important:** Do NOT share your `.env` file or commit it to Git!

## Step 5: Install Dependencies

```bash
pip install -r requirements.txt
```

This installs:
- `supabase==2.3.5` — Supabase Python client
- `python-dotenv==1.0.0` — Load environment variables from .env

## Step 6: Run the System

```bash
python3 face_attendance_system.py
```

The system will:
- Load credentials from `.env`
- Attempt to connect to Supabase
- If successful: Log attendance to Supabase
- If connection fails: Fall back to local CSV files

## How It Works

### Local Storage (Always)
- Student encodings → `data/student_records.csv`
- Enrollment photos → `data/snapshots/`

### Remote Storage (Supabase - If Enabled)
- Attendance records → `attendance` table in Supabase
- Automatic timestamp → Server-side timestamp for accuracy
- Cooldown protection → 5-minute window per student (configured locally)

### Columns in Attendance Table

| Column | Type | Description |
|--------|------|-------------|
| `id` | BIGINT | Unique record ID (auto-generated) |
| `name` | TEXT | Student name |
| `roll_no` | TEXT | Student roll number |
| `department` | TEXT | Student section/department |
| `timestamp` | TIMESTAMP | When attendance was logged |
| `created_at` | TIMESTAMP | Server creation time |

## Viewing Attendance Data

In Supabase dashboard:

1. Click **Table Editor** (left sidebar)
2. Select `attendance` table
3. View all logged records
4. Click on any row to see details

## Querying Attendance Data

### Example: View today's attendance

```sql
SELECT name, roll_no, department, timestamp
FROM attendance
WHERE DATE(timestamp) = CURRENT_DATE
ORDER BY timestamp DESC;
```

### Example: Count students present

```sql
SELECT COUNT(DISTINCT roll_no) as present_count
FROM attendance
WHERE DATE(timestamp) = CURRENT_DATE;
```

### Example: Export to CSV (in Supabase)

1. Table Editor → `attendance` table
2. Click the three dots menu
3. Select "Download as CSV"

## Troubleshooting

### "Could not connect to Supabase"
- Check your `.env` file has correct credentials
- Verify your Project URL and API key in Supabase dashboard
- Ensure your Supabase project is active (not paused)

### "Table attendance does not exist"
- Run the SQL schema in Supabase SQL Editor
- Verify the table name is exactly `attendance` (lowercase)

### System logs to CSV instead of Supabase
- Check if Supabase credentials are in `.env`
- Check the terminal output for error messages
- Attendance will still be recorded locally as fallback

### Attendance logged twice
- Check the 5-minute cooldown (`ATTENDANCE_COOLDOWN_MIN` in config.py)
- Each student can only be logged once every 5 minutes

## Security Notes

1. **Never commit `.env` to Git:**
   ```bash
   echo ".env" >> .gitignore
   ```

2. **Rotate API keys periodically** in Supabase dashboard

3. **Use Row Level Security (RLS)** in Supabase for production:
   - Table Editor → `attendance` table → RLS
   - Set policies to restrict data access

4. **Keep service_role secret key private** (don't share it)

## Advanced: Row Level Security (Optional)

For production systems, enable RLS:

```sql
-- Enable RLS on attendance table
ALTER TABLE attendance ENABLE ROW LEVEL SECURITY;

-- Allow inserts from app
CREATE POLICY "Allow service role to insert"
ON attendance
FOR INSERT
WITH CHECK (true);

-- Allow reads from app
CREATE POLICY "Allow service role to read"
ON attendance
FOR SELECT
USING (true);
```
